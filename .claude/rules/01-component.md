# 컴포넌트 규칙

## 기본 원칙

- 새 컴포넌트를 만들기 전에 기존 `frontend/src/components/`를 먼저 확인합니다.
- 페이지 전용 로직은 페이지 안에 두고, 두 곳 이상에서 재사용되면 컴포넌트로 분리합니다.
- 컴포넌트 파일명은 `PascalCase.tsx`를 사용합니다.
- props 타입은 컴포넌트 가까이에 명확히 정의합니다.
- 요청받지 않은 추상화나 공통화는 하지 않습니다.

## 버튼

- 모든 일반 버튼에는 가능한 경우 `type="button"`을 지정합니다.
- 아이콘이 있는 버튼은 `lucide-react` 아이콘을 우선 사용합니다.
- 아이콘 전용 버튼은 `aria-label`을 반드시 둡니다.
- 비활성 상태는 `disabled` 속성과 시각적 스타일을 함께 적용합니다.

## 모달

- 기존 `AuthModal`, `GoalModal`, `UploadModal`, `ProfileModal` 패턴을 우선 따릅니다.
- 모바일에서 화면 밖으로 넘치지 않게 높이와 스크롤을 확인합니다.
- 닫기 버튼, 배경 클릭, Escape 처리 여부는 기존 모달 패턴과 맞춥니다.

## 지도 관련 UI

- Leaflet 마커, 팝업, 툴팁은 레이어 정리 로직과 함께 구현합니다.
- 마커 HTML 문자열에 사용자 입력 또는 외부 API 값을 넣을 때는 XSS 가능성을 고려합니다.
- 모바일과 데스크톱의 CCTV 표시 방식이 다르면 상태 동기화를 확인합니다.

## 문서화

새 공통 컴포넌트가 생기면 이 문서에 이름, 역할, 주요 props를 추가합니다.
