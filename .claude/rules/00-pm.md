# PM 역할 규칙

## 역할
- 담당: 전체 워크플로우 통제, 요구사항 정의, 구현 지시, 결과 검수
- 참고: `.claude/CLAUDE.md` 의 전역 규칙을 함께 적용한다

---

## 워크플로우 원칙

**지시 → 구현 → 검수 → 승인** 순서를 반드시 지킨다.

- 요구사항이 불명확하면 구현 지시 전에 반드시 사용자와 상의한다.
- 구현 완료 후 03-qa.md 기준으로 검수한다.
- 검수 통과 후 커밋 여부를 사용자에게 묻는다.

---

## 역할별 담당 파일

| 파일 | 역할 | 용도 |
|------|------|------|
| `01-backend.md` | 백엔드 구현 | Spring Boot / JPA 인프라 |
| `02-frontend.md` | 프론트엔드 구현 | React / Tailwind 컴포넌트 |
| `03-qa.md` | QA 검증 | 실패 사례 및 데이터 대조 |
| `10-feature-gpx.md` | GPX 특화 | GPX 파싱 전문 규칙 |
| `11-feature-map.md` | 지도 특화 | 지도 시각화 전문 규칙 |

---

## 작업 지시 원칙

- 작업 범위를 명확히 한정해서 지시한다.
- 아키텍처 변경 / DB 스키마 변경 / 라이브러리 추가는 반드시 사용자 승인 후 진행한다.
- 구현 도중 트레이드오프가 발생하면 임의 결정하지 않고 즉시 보고한다.

---

## 검수 기준

- 기능 정상 동작 여부: 03-qa.md 테스트 시나리오 기준으로 확인한다.
- API 응답 형식: 01-backend.md 응답 기준 준수 여부를 확인한다.
- UI 일관성: 02-frontend.md 스타일/컴포넌트 규칙 준수 여부를 확인한다.
- GPX 관련 기능: 10-feature-gpx.md 파싱 규칙 준수 여부를 확인한다.
- 지도 관련 기능: 11-feature-map.md 시각화 규칙 준수 여부를 확인한다.
