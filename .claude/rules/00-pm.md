# PM / 작업 진행 규칙

## 역할

- 요청을 기능 단위로 나누고 백엔드, 프론트엔드, QA 관점의 영향을 확인한다.
- 불명확한 요구사항은 코드와 설정에서 먼저 추론한다.
- 위험한 변경, 데이터 삭제, 스키마 변경, 외부 서비스 설정 변경은 사용자 확인 후 진행한다.
- 사용자는 한국어로 짧고 직접적인 응답을 선호한다.
- 사용자가 "수정해"처럼 실행을 요청하면 계획만 말하지 말고 실제 파일을 수정한다.

## 진행 순서

1. 관련 파일과 현재 git 상태를 확인한다.
2. 변경 범위를 작게 정한다.
3. 기존 구조와 네이밍을 따른다.
4. 구현 후 빌드, 테스트, 린트 중 가능한 검증을 수행한다.
5. 최종 응답에는 변경 파일, 검증 결과, 남은 리스크만 간결히 보고한다.

## 금지 사항

- 요청받지 않은 대규모 리팩터링
- 사용자가 만든 변경사항 되돌리기
- 임의 의존성 추가
- 비밀키, 토큰, 개인 설정 파일 커밋
- 근거 없는 추측으로 API 계약 변경
- 문서 작업 중 앱 코드 변경사항을 정리하거나 되돌리기

## Codex 작업 메모

- 파일 검색은 `rg` 또는 `rg --files`를 우선 사용한다.
- 수동 편집은 `apply_patch`를 사용한다.
- 사용자가 커밋을 요청하기 전에는 커밋하지 않는다.
- 작업 중인 변경사항이 있으면 보존하고, 충돌 가능성이 있을 때만 사용자에게 묻는다.
- 현재 작업 트리에 CCTV 연동 변경사항이 있으므로, CCTV 작업을 직접 요청받기 전에는 해당 파일을 건드리지 않는다.
- `.claude/settings.json`과 `.claude/hooks/`는 Claude Code 호환용이다. Codex는 이를 자동 적용 설정으로 오해하지 않는다.
