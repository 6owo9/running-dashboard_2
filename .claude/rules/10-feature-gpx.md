# GPX 기능 규칙

## 범위

- GPX 파일 업로드
- 경로 좌표 파싱
- 거리, 시간, 페이스 계산
- 기록 저장 및 삭제

## 백엔드

- 업로드는 `multipart/form-data`를 사용한다.
- 허용 파일 크기는 설정의 multipart 제한을 따른다.
- 파일 원본은 파싱 후 저장하지 않는 것을 기본으로 한다.
- 좌표 배열은 프론트 지도 표시와 호환되는 순서를 유지한다.
- 잘못된 GPX, 빈 트랙, 좌표 누락은 명확히 처리한다.

## 프론트엔드

- 업로드 전 로그인 여부를 확인한다.
- 업로드 실패 시 사용자에게 메시지를 보여준다.
- 새 기록 업로드 후 목록, 통계, 지도 경로가 함께 갱신되어야 한다.
- 삭제 후에도 선택된 경로, 지도 bounds, 통계가 어긋나지 않아야 한다.

## QA 시나리오

- 정상 GPX 업로드
- 빈 GPX 또는 잘못된 XML 업로드
- 로그인 없이 업로드 시도
- 기록 삭제 후 새로고침
- 여러 기록이 있을 때 특정 기록 포커스
