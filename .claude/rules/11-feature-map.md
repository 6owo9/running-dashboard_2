# 지도 시각화 전문가 규칙

## 역할
- 담당: Leaflet.js 기반 지도 렌더링 및 경로 시각화
- 참고: `.claude/CLAUDE.md` 의 전역 규칙과 `02-frontend.md` 의 프론트엔드 규칙을 함께 적용한다

---

## 지도 라이브러리 규칙

- 지도 라이브러리는 **Leaflet.js** 만 사용한다. 다른 지도 라이브러리 사용 금지.
- 타일: 오픈스트리트맵 (`https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png`). 별도 API 키 불필요.
- GPX 파싱은 백엔드에서 처리한다. 프론트는 API 응답(좌표 배열)을 받아 지도에 표출한다.

---

## 경로 시각화 규칙

- 러닝 경로는 **폴리라인(Polyline)** 으로 표출한다.
- 경로 시작점에는 **원형 마커(CircleMarker)** 를 표출한다.
- 여러 경로를 동시에 표출할 경우, 모든 경로를 포함하는 범위로 지도를 자동 조정한다(`fitBounds`).
- 특정 날짜 경로로 이동 시 `fitBounds`를 사용한다.

---

## 지도 상태 관리

- 기간 전환(오늘 / 일주일) 중에는 로딩 오버레이를 표출한다.
- API 오류 발생 시 에러 메시지 오버레이를 표출한다.
- 경로 데이터가 없을 때 빈 지도가 아닌 안내 메시지를 표출한다.

---

## 지도 컴포넌트 규칙

- 지도 컴포넌트는 `MapPage.tsx` 에서 관리한다.
- 지도 인스턴스는 컴포넌트 언마운트 시 반드시 `map.remove()`로 정리한다.
- React StrictMode 환경에서 지도가 중복 초기화되지 않도록 ref로 초기화 여부를 관리한다.
