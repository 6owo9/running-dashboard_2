# QA 역할 규칙

## 역할
- 담당: 기능 검증, 오류 판단, 테스트 시나리오 작성 및 검수
- 참고: `.claude/CLAUDE.md` 의 전역 규칙을 함께 적용한다

---

## 기본 원칙
- 오류인지 아닌지 애매할 경우 **반드시 사용자에게 먼저 보고**하고 판단을 구한다.
- 오류가 아니라고 판단한 근거를 명확히 설명한다.
- 재현 가능한 오류만 오류로 판정한다. 한 번만 발생한 경우 재현 시도 후 판정한다.

---

## 오류 판정 기준

| 등급 | 기준 | 예시 |
|------|------|------|
| 🔴 Critical | 핵심 기능 동작 불가 | GPX 업로드 실패, 지도 미표출 |
| 🟠 Major | 기능은 동작하나 결과가 틀림 | 거리 계산 오류, 달성률 오표시 |
| 🟡 Minor | 기능은 정상이나 UX 문제 | 로딩 표시 누락, 에러 메시지 미표시 |
| 🟢 Normal | 오류 아님 | 의도된 동작, 디자인 차이 |

---

## 테스트 시나리오

### 🗺️ 지도 화면
| 항목 | 기대 결과 |
|------|----------|
| GPX 업로드 후 지도 진입 | 러닝 경로 폴리라인 표출 |
| 오늘 토글 선택 | 오늘 날짜 경로만 표출 |
| 일주일 토글 선택 | 최근 7일 경로 표출 |
| 기록 없을 때 | 빈 화면 대신 안내 메시지 표출 |
| 기간 전환 중 | 로딩 오버레이 표출 |
| API 오류 발생 시 | 에러 메시지 오버레이 표출 |
| 캘린더/기록 목록에서 날짜 클릭 | 해당 날짜 경로로 지도 fitBounds 이동 |
| 폴리라인 표출 | 경로 시작점에 원형 마커 표출 |
| 여러 경로 동시 표출 | 모든 경로를 포함한 범위로 지도 자동 조정 |

### 📁 기록 업로드 화면
| 항목 | 기대 결과 |
|------|----------|
| GPX 파일 업로드 | 업로드 성공 + 거리/시간/페이스 요약 카드 표출 |
| 잘못된 확장자 업로드 | 에러 메시지 표출, 업로드 중단 |
| 10MB 초과 파일 업로드 | 용량 초과 에러 표출, 업로드 중단 |
| 동일 파일명 재업로드 | alert 표출 후 업로드 중단 |
| 드래그 앤 드롭 업로드 | 클릭 업로드와 동일하게 동작 |
| 오늘 업로드 완료 시 | 헤더에 "오늘 완료" 뱃지 표출 |
| 업로드 중 | "업로드 중..." 표시 + 파일 선택 비활성화 |
| 기록이 있는 날짜 클릭 | 해당 날짜 기록이 파란색 하이라이트, 지도 연동 |
| 캘린더 월 이동 | 이전/다음 달 정상 이동, 기록 dot 표시 유지 |
| durationSeconds 없는 기록 | 요약 카드 시간/페이스 항목에 "-" 표출 |
| 기록 목록 없을 때 | "아직 러닝 기록이 없습니다." 안내 표출 |
| 기록 목록 로딩 중 | "불러오는 중..." 표시 |
| 기록 목록 API 오류 | 에러 메시지 표출 |

### 🎯 목표치 화면
| 항목 | 기대 결과 |
|------|----------|
| 목표 미설정 시 | 입력 폼이 바로 열림 (안내 메시지 없이) |
| 목표 거리 입력 후 저장 클릭 | 확인 절차 박스(amber) 표출 |
| 확인 절차에서 "확인" 클릭 | 목표 저장 + 프로그레스 바 업데이트 |
| 확인 절차에서 "취소" 클릭 | 입력 폼으로 복귀 |
| 0 이하 또는 빈 값 입력 후 저장 | "올바른 거리(km)를 입력해주세요." 에러 표출 |
| "수정" 버튼 클릭 | 기존 목표값이 입력 필드에 pre-fill |
| 달성률 100% 초과 | 프로그레스 바 최대 100% 고정 |
| 목표 달성 시 | "100%" 표시 |
| 저장 중 | "저장 중..." 버튼 비활성화 |
| 목표 로딩 중 | "불러오는 중..." 표출 |
| 목표 API 오류 | 에러 메시지 표출 |

### 🔌 API
| 항목 | 기대 결과 |
|------|----------|
| `GET /api/running-records` | 200 OK + 전체 기록 배열 반환 |
| `GET /api/running-records?period=today` | 200 OK + 오늘 기록만 반환 |
| `GET /api/running-records?period=week` | 200 OK + 최근 7일 기록 반환 |
| `GET /api/running-records?period=invalid` | 400 반환 |
| 기록 없을 때 기간 조회 | 200 OK + 빈 배열 `[]` 반환 (404 아님) |
| `POST /api/running-records/upload` GPX | 200 OK + 저장된 기록 반환 |
| trkpt 0개인 GPX 업로드 | 400 반환 |
| 잘못된 확장자 업로드 | 400 반환 |
| `POST /api/goals` | 200 OK + 저장된 목표 반환 |
| `GET /api/goals/current` | 200 OK + 현재 목표 및 달성률 반환 |
| `GET /api/goals/current` (목표 미설정) | 200 OK + `data: null` 반환 |
| `GET /api/stats/summary` | 200 OK + 전체 통계 반환 |
| 서버 재시작 후 | data.sql 데이터 정상 로드 |
