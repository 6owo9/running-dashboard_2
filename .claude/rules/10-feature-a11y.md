# 접근성 규칙

## 기본 원칙

- 키보드만으로 주요 액션에 접근할 수 있어야 합니다.
- 아이콘 전용 버튼에는 `aria-label`을 둡니다.
- 이미지에는 의미 있는 `alt`를 둡니다.
- 색상만으로 상태를 전달하지 않습니다.

## 모달

- 닫기 버튼은 명확한 이름을 가져야 합니다.
- 모바일 전체 화면 모달에서도 포커스 가능한 닫기 버튼이 있어야 합니다.
- 배경과 본문 대비가 충분해야 합니다.

## 지도

- 지도 자체는 보조 정보일 수 있으므로, 주요 기록 데이터는 목록이나 텍스트로도 확인 가능해야 합니다.
- 마커 클릭으로만 접근 가능한 정보는 모바일 모달 또는 목록 대체 경로를 고려합니다.
- CCTV 영상이 없거나 실패하면 텍스트 fallback을 제공합니다.

## 폼

- 입력 필드에는 label 또는 접근 가능한 이름을 제공합니다.
- 오류 메시지는 사용자에게 직접 노출합니다.
- 로그인, 회원가입, 프로필 변경 폼은 submit 실패 시 원인을 보여줍니다.

## QA 체크

- Tab 이동 순서가 자연스러운지 확인합니다.
- 버튼 이름이 스크린리더에 의미 있게 전달되는지 확인합니다.
- 모바일 화면에서 닫기/취소 버튼이 가려지지 않는지 확인합니다.
