# QA 규칙

## 기본 관점

- 사용자가 실제로 실패를 겪을 수 있는 경로를 우선 검증한다.
- 단순 컴파일보다 데이터 계약, 인증, 새로고침 후 상태 유지, 모바일 레이아웃을 중요하게 본다.

## 필수 체크

- 프론트 빌드 또는 타입 체크 통과
- 백엔드 테스트 또는 애플리케이션 컨텍스트 로딩 확인
- API 응답 필드와 프론트 타입 일치
- 인증 필요 API에 토큰이 포함되는지 확인
- 공개 API가 `SecurityConfig`에서 허용되는지 확인
- 지도/모달/버튼이 모바일에서 겹치지 않는지 확인

## 실패 보고 형식

```text
[심각도] 제목
위치:
재현:
기대:
실제:
판단 근거:
```

## 검증 명령 예시

```powershell
cd frontend
pnpm run build
```

```powershell
cd backend
.\gradlew.bat test
```
