# 스타일 규칙

## 기본 방향

- 기존 Tailwind 클래스와 프로젝트 색상 사용 방식을 우선 따릅니다.
- 인라인 스타일은 Leaflet 팝업처럼 라이브러리 제약이 있는 경우에만 제한적으로 사용합니다.
- 임의의 대규모 디자인 변경은 하지 않습니다.
- 모바일과 데스크톱에서 텍스트가 버튼, 카드, 모달 밖으로 넘치지 않게 합니다.
- 이 프로젝트는 템플릿처럼 `ct-*` 클래스와 별도 `tailwind.css` 토큰을 강제하는 구조가 아닙니다. 템플릿 규칙은 참고하되, 현재 코드의 일반 Tailwind 유틸리티 중심 방식을 우선합니다.

## 금지 패턴

| 금지 | 대신 사용 |
| --- | --- |
| 이유 없는 임의 hex 색상 추가 | 기존 색상 또는 명확한 토큰 |
| `z-[9999]` 남발 | 필요한 위치에서만 제한적으로 사용 |
| 같은 UI의 중복 스타일 | 컴포넌트 또는 공통 클래스 |
| 장식 목적의 과한 그라디언트 | 기능 중심의 절제된 스타일 |
| 템플릿용 `ct-*` 클래스 무리한 도입 | 기존 Tailwind 패턴 유지 |

## 레이아웃

- 지도, 통계, 기록 목록이 한 화면 안에서 겹치지 않도록 반응형 기준을 확인합니다.
- 고정 위치 버튼은 Leaflet 컨트롤, 모달, 모바일 브라우저 UI와 겹치지 않게 배치합니다.
- 카드 안에 다시 장식용 카드를 중첩하지 않습니다.

## 타이포그래피

- 대시보드 내부 텍스트는 스캔하기 쉬운 크기와 굵기를 사용합니다.
- 히어로 페이지가 아니므로 과도하게 큰 제목을 만들지 않습니다.
- 버튼 텍스트는 줄바꿈 또는 축약을 고려합니다.

## 이미지와 아이콘

- 버튼이나 도구에는 가능한 경우 `lucide-react` 아이콘을 사용합니다.
- 실제 상태를 보여줘야 하는 영역에는 장식 이미지보다 실제 데이터 표시를 우선합니다.
- 이미지에는 목적에 맞는 `alt`를 지정합니다. 장식이면 `alt=""`를 사용합니다.

## Leaflet 팝업 스타일

- 팝업 HTML은 최소화합니다.
- 영상, 이미지, 외부 URL이 실패할 경우 빈 영역 대신 fallback을 보여줍니다.
- 마커 활성 상태는 색상, 팝업 열림 상태, 선택 상태가 일치해야 합니다.
- 외부 API에서 온 이름이나 URL을 HTML 문자열에 넣을 때는 escaping 또는 안전한 렌더링 방식을 고려합니다.
